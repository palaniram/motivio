Build a fake door landing page for a micro SaaS product called MotivIO.

## What MotivIO Is
A motivated seller list tool for Bay Area real estate investors. The core promise:
get a clean, enriched, ready-to-work motivated seller list in 10 minutes — no
setup, no learning curve, no vendor dependency. Bay Area counties only (Alameda,
Contra Costa, San Mateo). Pre-foreclosure and tax-delinquent leads. Everything
pre-configured. Works in 10 minutes from signup to list in hand.

## Purpose of This Page
This is a fake door — there is no product yet. The goal is to validate whether
Bay Area investors will sign up for a waitlist before we build anything. We are
measuring: does the headline stop them, does the promise resonate, will they
give us their email.

## Target Visitor
Bay Area real estate investor doing 1-5 deals per month. Currently either:
- Paying a vendor for motivated seller lists and unhappy with quality
- Using PropStream but finding it too complex and time-consuming
- Manually pulling county records which takes hours
They are not data experts. They want the list, not the tool.

## Design Requirements

Typography:
- Headings: Inter or similar clean sans-serif, bold weight
- Body: Same family, regular weight, comfortable line height
- No serif fonts anywhere

Colors:
- Primary: #0B7A75 (teal) — CTAs, highlights, nav
- Navy accent: #0D2137 — section headings, footer
- Gold accent: #C8902A — comparison table highlight, step numbers
- Background: #FFFFFF primary, #F7F8FA for alternating sections
- Text: #2C3E50 body, #5A6A7A secondary
- Borders and dividers: #E0E4EA

Spacing and layout:
- Full width sections with generous padding (80–100px vertical)
- Content max-width: 1100px, centered
- Cards: subtle shadow, 8px border radius, white background
- No heavy gradients, no dark backgrounds except footer
- Clean, professional, minimal decoration

CTA buttons:
- Background: #0B7A75
- Text: white, bold
- Border radius: 6px
- Padding: 14px 28px
- Hover: slightly darker teal (#085E5A)

Comparison table:
- MotivIO column: teal header (#0B7A75), light teal rows (#E6F4F3)
- Checkmark: ✓ in green (#1A7A4A)
- X mark: ✗ in red (#9B2335)
- Clean borders, alternating row shading for other columns

## Functional Requirements

Waitlist form behavior:
- Email input + submit button (appears in hero and final CTA section)
- On submit: show a thank you message inline — do not redirect
- Thank you message: "You're on the list. We'll reach out personally before
  launch with your free trial access."
- Store submitted emails in localStorage for now (no backend needed yet)
- Basic email format validation before submission

Scroll behavior:
- Smooth scroll when nav CTA clicks to hero form
- Page should feel fast and lightweight

Mobile:
- Fully responsive
- Three-column layouts collapse to single column on mobile
- Comparison table scrolls horizontally on small screens
- CTA buttons full width on mobile

## Technical Requirements
- Single HTML file with embedded CSS and vanilla JavaScript
- No frameworks, no dependencies, no external libraries
- Google Fonts import for Inter (or system font fallback)
- No backend, no API calls, no tracking pixels
- File should be self-contained and openable directly in a browser
- Also deployable to Netlify Drop or Vercel with zero configuration

Produce a single complete index.html file ready to open in a browser or
